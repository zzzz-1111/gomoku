# 墨弈 Gomoku

墨弈是一款基于 Qt Widgets 开发的桌面五子棋程序，支持本地对战、人机对战、TCP 联机对战、用户头像昵称、历史记录、棋局复盘和对局动画展示。

## 功能概览

- 本地双人对战
- 人机对战
  - 支持难度选择
  - 支持玩家选择黑方或白方
- TCP 联机对战
  - 点击“联机对战”后选择作为主机或客户端
  - 主机监听客户端连接，并显示可提供给客户端输入的地址
  - 客户端输入主机 IP 或 `IP:端口` 后连接
  - 主机未检测到客户端接入前不能落子
  - 对局结束后由主机决定是否再来一局
- 用户信息
  - 首次运行时输入昵称
  - 设置页可修改昵称
  - 设置页可上传头像
  - 联机和人机对战中显示双方昵称与头像
- 视觉效果
  - 开局双方头像入场动画
  - VS 图片展示
  - 黑方或白方胜利图片动画
- 历史记录
  - 保存本地、人机、联机对局结果
  - 记录黑方、白方、胜者、总步数和对局时间
- 棋局复盘
  - 从历史记录中打开复盘
  - 按落子顺序回放棋盘
  - 默认每手间隔 `0.3s`

## 运行环境

推荐环境：

- Windows
- Qt 6.11.1
- MinGW 64-bit
- Qt Creator

## 构建方式

推荐使用 Qt Creator 打开项目根目录下的 `CMakeLists.txt`，选择 Qt 6 MinGW 套件后构建运行。

命令行构建示例：

```bash
cmake -S . -B build
cmake --build build --config Release
```

如果使用 Qt Creator 生成的构建目录，也可以在对应 build 目录中执行：

```bash
cmake --build . --config Release
```

## 联机使用说明

### 局域网联机

1. 主机点击“联机对战”。
2. 选择“作为主机”。
3. 选择主机执黑或执白。
4. 程序进入对局界面，并弹出主机地址，例如 `192.168.1.8:5000`。
5. 客户端点击“联机对战”。
6. 选择“作为客户端”。
7. 输入主机提供的地址。

同一台电脑本地测试时，客户端可以输入：

```text
127.0.0.1:5000
```

### 内网穿透联机

主机本地默认监听端口是 `5000`。如果使用内网穿透，需要将外部地址映射到主机电脑的 `5000` 端口。

示例：

```text
穿透地址:外部端口 -> 主机电脑:5000
```

客户端应输入穿透服务提供的外部地址和外部端口，例如：

```text
abc.example.com:12345
```

## 默认配置

主要配置位于 `src/common/config.h`：

- `kBoardSize = 15`
- `kDefaultCellSize = 40`
- `kDefaultMargin = 24`
- `kDefaultPort = 5000`
- `kDefaultReplayDelayMs = 300`

## 模块说明

### `src/app`

程序入口，负责启动 Qt 应用和主窗口。

### `src/common`

公共类型和全局配置，例如棋盘大小、默认端口、复盘间隔等。

### `src/core`

对局核心逻辑，包括棋盘状态、落子规则、胜负判断、回合控制和玩家行动接口。

### `src/ai`

人机对战相关逻辑，包括 AI 落子、棋形评估和不同难度策略。

### `src/network`

联机通信模块。

- `game_server.*`：主机监听、客户端接入、消息广播。
- `network_manager.*`：客户端连接主机、发送和接收消息。
- `protocol.*`：网络消息类型和 JSON 序列化。

### `src/profile`

本地用户信息管理，包括昵称、默认头像、自定义头像和网络头像编码。

### `src/record`

对局历史记录管理，负责保存棋局和落子记录。

### `src/data`

数据库基础管理和数据模型。

### `src/ui`

界面相关模块，包括主窗口、棋盘控件、历史记录窗口、复盘窗口和开局动画层。

## 资源说明

资源文件位于 `res/assets`：

- `avatars/`：默认头像、AI 头像、黑白方头像
- `backgrounds/`：首页背景、对局背景、VS 图片
- `board/`：棋盘、棋子、胜利图片
- `icons/`：程序图标

## 当前说明

- 棋盘默认是 15 路。
- 联机模式采用 TCP 直连，不包含房间列表和局域网房间扫描。
- 客户端可以输入 `IP` 或 `IP:端口`。
- 不写端口时默认使用 `5000`。
- 联机重开由主机决定，客户端被动同步。
- 历史记录和复盘属于本地数据，保存在本机数据库中。

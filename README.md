# 《墨弈》功能文档

**文档版本：** V1.0  
**开发语言：** C++  
**界面框架：** Qt  
**网络方式：** TCP 联机通信  
**数据存储：** SQLite  
**人工智能方案：** 启发式评分 + Minimax/Alpha-Beta 剪枝  
**人工智能参考：** 《Go-Moku and Threat-Space Search》（L.V. Allis, H.J. van den Herik, M.P.H. Huntjens）  
**计划周期：** 5天

---

## 一、项目概述

本项目拟开发一套基于 Qt 的智能五子棋竞技系统“墨弈”。系统不只是实现普通五子棋，而是将联机对战、人机对战、棋局记录和棋谱回放结合起来。

系统主要提供三种使用方式：

1. 本地双人对战；
2. 与本地人工智能对战；
3. 联机环境下两台客户端实时对战。

项目采用客户端/服务器架构。服务器负责玩家连接、回合控制、落子合法性验证和棋局状态同步；客户端负责棋盘绘制、用户交互和记录展示。

---

## 二、项目目标

系统需要达到以下目标：

- 完成一套可运行、可演示的五子棋程序；
- 支持联机双人实时对战；
- 支持不同风格和难度的本地人工智能；
- 支持保存每一步棋并进行棋谱回放；
- 支持对局结果和玩家战绩统计；
- 能在答辩中展示 Qt、网络通信、游戏逻辑、算法和数据库等技术点。

---

## 三、用户角色

### 1. 普通玩家

可以进行本地对战、人机对战和联机对战，并查看自己的历史战绩。

### 2. 房主

在联机模式下创建房间，等待其他玩家加入，并设置先后手、棋盘大小和回合时间。

### 3. 对战玩家

加入房间后，与房主进行实时对局。

### 4. 人工智能玩家

由本地人工智能引擎控制，能够根据当前棋局计算下一步落点。

## 五、详细功能需求

### 5.1 主界面

主界面用于选择游戏模式和进入其他功能页面。

应包含：

- 本地双人对战；
- 人机对战；
- 联机对战；
- 历史棋局；
- 玩家战绩；
- 系统设置；
- 退出系统。

主界面显示系统名称、当前玩家昵称和累计对局数据。

### 5.2 棋盘显示

棋盘采用自定义 QWidget 绘制。

主要功能：

- 绘制标准 15×15 棋盘；
- 绘制黑棋和白棋；
- 显示鼠标悬停位置；
- 高亮显示最后一步棋；
- 标记推荐落点；
- 在游戏结束时显示五连线；
- 窗口缩放时自动调整棋盘尺寸。

棋盘坐标应根据控件大小动态计算，避免使用大量固定坐标。

### 5.3 落子控制

用户点击棋盘交叉点后，系统将鼠标位置转换为棋盘坐标。

系统需要检查：

- 点击位置是否在棋盘范围内；
- 当前交叉点是否已有棋子；
- 是否轮到当前玩家；
- 当前游戏是否已经结束；
- 网络模式下服务器是否确认该操作。

合法落子后，更新棋盘并切换当前玩家。

### 5.4 胜负判断

每次落子后，从最后落子位置检查四个方向：

- 水平方向；
- 垂直方向；
- 左上至右下；
- 右上至左下。

某一方向连续同色棋子数量达到5时，判定该玩家获胜。

系统还应支持：

- 棋盘下满后的和棋判断；
- 对局结束后禁止继续落子；
- 显示获胜玩家；
- 高亮构成五连的棋子。

### 5.5 本地双人模式

两个玩家在同一台电脑上轮流操作。

功能包括：

- 黑白双方轮流落子；
- 显示当前回合；
- 显示本局步数；
- 重新开始；
- 保存棋局；
- 对局结束后生成记录。

本地双人模式不需要服务器。

### 5.6 人机对战模式

玩家与本地人工智能进行对战。

玩家可以设置：

- 玩家执黑或执白；
- 人工智能难度；
- 人工智能性格；
- 是否显示推荐落点。

#### 人工智能难度

| 难度 | 实现方式                     |
| ---- | ---------------------------- |
| 简单 | 在高评分候选点中随机选择     |
| 普通 | 启发式评分并搜索较少层数     |
| 困难 | Alpha-Beta剪枝并增加搜索深度 |

#### 人工智能性格

| 性格   | 行为特点               |
| ------ | ---------------------- |
| 进攻型 | 更重视形成活三、活四   |
| 防守型 | 更重视阻止玩家形成威胁 |
| 平衡型 | 综合考虑进攻和防守     |

人工智能不连接外部大模型，也不依赖云端接口，全部在本机计算。

### 5.7 人工智能提示功能

游戏过程中，玩家可以点击“智能提示”。

系统分析当前棋局后：

- 标记推荐落点；
- 显示该位置的评分；
- 给出简短原因；
- 显示当前主要威胁。

示例：

```text
推荐落点：H8
局面评分：8600
分析结果：该位置可阻止对方形成活四，同时保留己方进攻机会。
```

该功能是辅助分析，不自动替玩家落子。

### 5.8 联机对战

系统采用 C/S 架构。

```text
客户端A ──TCP── 游戏服务器 ──TCP── 客户端B
```

#### 创建房间

房主可以：

- 启动本地服务器；
- 设置监听端口；
- 创建房间；
- 设置昵称；
- 设置先后手；
- 等待另一名玩家连接。

#### 加入房间

另一名玩家输入：

- 服务器地址；
- 端口号；
- 玩家昵称。

连接成功后进入对局。

#### 网络同步

服务器负责：

- 分配黑白双方；
- 控制当前回合；
- 验证落子是否合法；
- 保存权威棋盘状态；
- 将落子广播给双方；
- 判断胜负；
- 同步重新开始请求。

客户端不能直接决定对局结果，只负责提交操作和显示服务器返回的状态。

### 5.9 网络消息类型

系统可以使用 JSON 或自定义文本协议传输消息。

| 消息类型        | 作用           |
| --------------- | -------------- |
| LOGIN           | 玩家进入服务器 |
| CREATE_ROOM     | 创建房间       |
| JOIN_ROOM       | 加入房间       |
| READY           | 玩家准备       |
| GAME_START      | 开始游戏       |
| MOVE            | 提交落子       |
| MOVE_RESULT     | 返回落子结果   |
| RESTART_REQUEST | 请求重新开始   |
| GAME_OVER       | 游戏结束       |
| DISCONNECT      | 玩家断开连接   |

示例：

```json
{
    "type": "MOVE",
    "x": 7,
    "y": 8,
    "player": "black"
}
```

需要处理 TCP 粘包和拆包问题，可以使用“消息长度 + 消息正文”的数据格式。

### 5.10 棋局记录

每一局游戏都记录以下信息：

- 对局编号；
- 游戏模式；
- 黑方名称；
- 白方名称；
- 开始时间；
- 结束时间；
- 获胜方；
- 总步数；
- 每一步的坐标和落子方。

落子记录示例：

```text
第1步：黑方 H8
第2步：白方 H9
第3步：黑方 I8
```

### 5.11 棋谱回放

用户可以从历史列表中选择一场比赛进行回放。

回放页面支持：

- 从头开始；
- 上一步；
- 下一步；
- 自动播放；
- 暂停；
- 调整播放速度；
- 跳转到指定步数；
- 显示当前步数和落子方。

回放过程中不允许直接修改棋盘。

### 5.12 胜率或局势趋势图

系统可以根据每一步的局面评分绘制趋势图。

横轴表示落子步数，纵轴表示局面优势：

```text
正数：黑方占优
负数：白方占优
```

该图不需要宣称是真实胜率，可命名为：

> 局势评分变化曲线

这样算法含义更准确，也更容易解释。

### 5.13 玩家战绩

系统统计：

- 总对局数；
- 胜利次数；
- 失败次数；
- 和棋次数；
- 胜率；
- 人机模式战绩；
- 联机模式战绩；
- 最近对局时间。

用户可以在主界面或战绩页面查看这些信息。

### 5.14 系统设置

系统设置包括：

- 玩家昵称；
- 棋盘背景；
- 棋子样式；
- 是否播放落子音效；
- 是否显示坐标；
- 默认服务器端口；
- 默认人工智能难度；
- 默认人工智能性格；
- 是否显示人工智能提示。

设置保存在本地配置文件或 SQLite 数据库中。

## 九、项目亮点

本项目的主要亮点不是单纯实现五子棋，而是将多个模块组合成一个完整系统：

- 使用 Qt 自定义绘制交互式棋盘；
- 采用 C/S 架构实现联机实时对战；
- 由服务器统一维护权威棋局状态；
- 实现本地启发式五子棋人工智能；
- 支持不同人工智能性格和难度；
- 保存完整棋谱和历史战绩；
- 支持自动回放和关键步骤分析；
- 通过局势评分曲线展示棋局变化。

项目最终可命名为：

> **墨弈**
